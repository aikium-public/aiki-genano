## Reproducing the smoke tests

Each generator has its own `generators/test_<tool>.py`. Prerequisites: a Python 3.12 venv with `torch==2.4.*+cu121`, `transformers==4.44.*`, `accelerate==0.33.*`, `bitsandbytes`, `fair-esm`, `biotite<0.40`, `torch-scatter` (matched to torch), `protobuf`, `biopython`.

```bash
# Generator that doesn't need HF auth
python generators/test_iglm.py            # weights bundled in the wheel

# Generators that need cached HF weights (no token required)
python generators/test_protgpt2.py
python generators/test_nanobert.py
python generators/test_esmfold.py

# Gated — need an HF token with read-on-gated-repos permission plus a
# manual access request at huggingface.co/Lab608/NanoAbLLaMA
HF_TOKEN=... python generators/test_nanoabllama.py
HF_TOKEN=... python generators/test_pepmlm.py     # click-through licence

# ProteinDPO — clone https://github.com/evo-design/protein-dpo, apply
# patches/proteindpo_sample.patch, download ESM-IF1 weights, then run the
# invocation in generators/test_proteindpo.py.

# IgGM — clone https://github.com/TencentAI4S/IgGM, apply env patches
# (sympy==1.12, openmm==8.1.*), then:
python generators/test_iggm.py            # needs antigen PDB from ESMFold prefold
```

## Targets

`targets.fasta` carries two test peptides used for smoke runs (`SLNFLGGLPPL`, an HIV-gp120-like 11mer, and `DIQMTQSPSSLSASVGDRVTITC`, a kappa FR1 fragment). Replace with your own epitope list for a full comparison run. The 10 representative GPCR target identifiers used in the mAbs (submitted) paper are in the Zenodo deposit at `targets_keys.csv`; the corresponding peptide-window sequences are not redistributed and are available via `partnerships@aikium.com` under NDA.

## ESMFold prefold + IgGM pipeline

IgGM needs a structural antigen PDB, not a peptide string. `fold_targets.py` is the prefolder: given any FASTA, it writes one PDB per record into `example_outputs/esmfold/`. Budget on a 16 GB GPU: ~1.3 s/target for peptides ≤40 AA, 8.5 GB peak VRAM. For 65 targets: ~2 min total.

IgGM itself runs at ~16 s/sample × 100 samples × 65 targets ≈ 29 h on one 16 GB GPU, or ~4 h on a 4×A100-40 node.

## What's not in this folder

- **Foundation-model weights.** Every tool's checkpoints are downloaded on first run (HuggingFace or Zenodo) or shipped with the pip wheel (IgLM). We don't redistribute weights.
- **Cloned competitor codebases.** ProteinDPO and IgGM live as separate clones on the user's disk; this folder ships invocation recipes and patches only.
- **Aiki-GeNano's own property predictors.** TEMPRO, NetSolP, Sapiens are wrapped in `aiki_genano/evaluation/`.
