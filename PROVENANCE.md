# Data Provenance

This document maps every paper figure to:
- The exact CSV(s) it was drawn from
- The notebook cell that generated it
- Any filters / derived columns applied

All source data is in `gs://gennano-gdpo-models/data/statistical/statistical/`.

> **Important note on NetSolP:** The `_all_profiled.csv` merged files are missing `netsolp_solubility` for SFT/DPO/GDPO because the merge was created before NetSolP was added to those runs. The individual `_seed{s}_temp{t}_profiled.csv` files at T=0.7 DO contain full NetSolP data (~100 seqs/target × 3 seeds). Always use the individual per-seed/temp files for solubility analysis.

All figures are generated by `analysis/generation_2.ipynb` (run from the `genseq2/` root).

## Profiling Pipeline — How Properties Were Computed

The property evaluation pipeline runs in two stages:

### Stage 1 — Local properties (`profile.py`)

```
genseq2/src/binder_design/protgpt2_dpo/analysis/profile.py
  └── compute_sequence_profile(seq) → dict of ~50 columns
```

Computes entirely locally (CPU, no external API):

| Property group | Columns | Method |
|---|---|---|
| GDPO reward scores | `reward_scaffold_integrity`, `reward_liability`, `reward_hydrophobic_patch`, `reward_fr2_aggregation`, `reward_vhh_hallmark`, `reward_expression` | Reward functions from `rewards/rewards.py` |
| Chemical liabilities | `liability_severity`, `motif_NG/NS/NT/NN/NH/NA/NQ`, `motif_DG/DS/DT/DD/DH/DP`, `motif_N_glyco`, `motif_RGD/NGR/LDV/CDR_Met` | Motif scanning in `profile.py` |
| Biophysical | `gravy`, `isoelectric_point`, `instability_index`, `aromaticity`, `molecular_weight`, `charge_at_pH7`, `charge_at_pH74` | Biopython `ProteinAnalysis` |
| Secondary structure | `helix_fraction`, `turn_fraction`, `sheet_fraction` | Biopython |
| CDR/FR regions | `cdr1_length`, `cdr2_length`, `cdr3_length`, `total_cdr_length`, `cdr1_charge`, `cdr2_charge`, `cdr3_charge`, `fr2_length` | IMGT boundaries in `rewards/cdr_utils.py` |
| Aggregation | `aggrescan_na4vss`, `aggrescan_nhs`, `aggrescan_aat` | Aggrescan3D scale (Conchillo-Solé 2007) |

### Stage 2 — External model properties (`run_properties.py`)

```
genseq2/src/binder_design/protgpt2_dpo/analysis/run_properties.py
  ├── run_tempro.py         → tempro_tm         (TEMPRO: ESM-2 + Keras)
  ├── run_sapiens_tempro.py → sapiens_humanness  (Sapiens humanness model)
  └── run_netsolp.py        → netsolp_solubility, netsolp_usability  (NetSolP-1.0)
```

These three add the final columns to produce the `_profiled.csv` files.

### Entry point

```bash
# Inside Docker, from genseq2/ root:
python -m src.binder_design.protgpt2_dpo.analysis.run_properties --model SFT
python -m src.binder_design.protgpt2_dpo.analysis.run_properties --model DPO
python -m src.binder_design.protgpt2_dpo.analysis.run_properties --model GDPO
python -m src.binder_design.protgpt2_dpo.analysis.run_properties --model GDPO_SFT
```
